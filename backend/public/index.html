API Documentation

Welcome to the API documentation of my project. Below is a list of available endpoints and their descriptions.

Events Endpoints

  • GET /api/events

    Retrieve a paginated list of events. Supports sorting and pagination.

    Query Parameters:

    • limit (optional): Number of events to return. Default: 10.
    • offset (optional): Number of events to skip. Default: 0.
    • sortBy (optional): Field to sort by (e.g., date, title).
    • sortOrder (optional): Sorting order (asc or desc).

    Example:

    GET /api/events?limit=5&offset=10&sortBy=title&sortOrder=asc

  • GET /api/events/:id

    Retrieve details of a specific event by its ID.

  • POST /api/events

    Create a new event.

    Body: JSON object containing title, description, date, and location.

  • PUT /api/events/:id

    Update an existing event by its ID.

    Body: JSON object with fields to update.

  • DELETE /api/events/:id

    Delete a specific event by its ID.

  • GET /api/events/search

    Search for events based on specific fields.

    Query Parameters:

    • title (optional): Search events by title.
    • location (optional): Search events by location.
    • date (optional): Search events by a specific date (format: YYYY-MM-DD).

Tickets Endpoints

  • GET /api/tickets

    Retrieve a paginated list of tickets. Supports sorting and pagination.

  • GET /api/tickets/:id

    Retrieve details of a specific ticket by its ID.

  • POST /api/tickets

    Create a new ticket for an event.

    Body: JSON object containing event (event ID), price, quantity, and status.

  • PUT /api/tickets/:id

    Update an existing ticket by its ID.

    Body: JSON object with fields to update.

  • DELETE /api/tickets/:id

    Delete a specific ticket by its ID.

Charities Endpoints

  • GET /api/charities

    Retrieve all charities.

  • POST /api/charities

    Create new charities. Accepts an array of charity objects.

    Body: JSON array containing objects with title, description, date, and location.

  • PUT /api/charities/:id

    Update an existing charity by its ID.

    Body: JSON object with fields to update.

  • DELETE /api/charities/:id

    Delete a specific charity by its ID.

  • GET /api/charities/search

    Search for charities based on specific fields.

    Query Parameters:

    • title (optional): Search charities by title.
    • location (optional): Search charities by location.
    • date (optional): Search charities by a specific date (format: YYYY-MM-DD).

Usage

Use tools like Postman or curl to test these endpoints. Ensure your server is running on http://localhost:3000 or the appropriate host.
